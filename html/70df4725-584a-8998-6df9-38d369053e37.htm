Context Class
Defines a context for expressions. This provides the expression with the source of top-level identifiers. There are two uses for the context class: - The evaluation or execution context. In this version, the context contains actual instances of classes, registered with RegisterObjectInstance(String, Object, Boolean). - The compiling context. Here actual object instances are not given, instead only type information is stored via RegisterObjectInstanceType(String, Type, IPropertyTyper, Boolean). For resolving possible ambiguous identifiers the first registered instance or type is used first. Because from this the order of registration matters, when using precompilation it is best to use the exact same registration order for both the compiling and evaluation context to avoid odd behaviour.
