Classes

BinaryExpression
Evaluates operators with two operands. Since this class is nearly equivalent to binary expressions, see that class for full explanations.
CallExpression
Evaluates a function call. Overloaded methods are supported.
Context
Defines a context for expressions. This provides the expression with the source of top-level identifiers. There are two uses for the context class: - The evaluation or execution context. In this version, the context contains actual instances of classes, registered with RegisterObjectInstance(String, Object, Boolean). - The compiling context. Here actual object instances are not given, instead only type information is stored via RegisterObjectInstanceType(String, Type, IPropertyTyper, Boolean). For resolving possible ambiguous identifiers the first registered instance or type is used first. Because from this the order of registration matters, when using precompilation it is best to use the exact same registration order for both the compiling and evaluation context to avoid odd behaviour.
ContextRegisteredObject
Support class. Contains the information for a single registered class / type.
EvaluationException
This exception is raised when a custom error occurs during execution. Not all possible exceptions are wrapped in this exception.
Expression
The basic expression node. All other expressions inherit from this class and implement its abstracts.
ExpressionT
A typed variation of Expression. Assumes the expression result can be cast to the given type parameter.
ExpressionType
This class is used within the parser/expression system to communicate the compile-time type of an expression. Outside the parser system you should never need to use this class.
ExpressionUtil
Provides a number of convenience methods for using the expression parser.
IndexedExpression
Evaluates indexing an instance, either because the instance is an array, or because it has an indexer method defined. Note that current indexing support is limited to one argument, so multidimensional arrays or multi-argument indexers are not supported. This implementation checks the DefaultMemberAttribute of a class for the name of the indexer, so indexers with non-default names work.
ParseException
This exception is raised when an error occurs during parsing. These errors are generally of the order of unexpected characters in the input, or incorrect tokens at some point.
Parser
QualifiedExpression
Evaluates an expression that qualifies a class instance or static type.
Tokenizer
Splits an input string into tokens. The NextToken method moves through the token stream, and the current token can be checked through Current, but generally it is more useful to use the except/accept set of methods. This is a support class for the Parser, you should never need to create an instance of this class yourself.
TokenizerToken
Contains all information about a single token. Based on the token's type, a different value field should be examined. The list is as follows: Bool: ValueBool Int: ValueInt Double: ValueDouble String, Identifier, Special: ValueString Null, End: None.
TrinaryExpression
Evaluates the trinary boolean select expression.
UnaryExpression
Evaluates operators with a single operand. Since this class is nearly equivalent to binary expressions, see that class for full explanations.
ValueExpression
Expression node for a single literal.

IPropertyHolder
An interface for classes that want to provide property values based on run-time information.
IPropertyTyper
An companion interface to IPropertyHolder to assist in parse-time optimizations and checks. A class that implements IPropertyHolder can also implement this interface, or it can be implemented on a different class. When making this decision consider if the IPropertyHolder class would have a valid instance at parse time.
